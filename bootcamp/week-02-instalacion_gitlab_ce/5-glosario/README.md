# Glosario — Semana 02

## B

- **Backup (gitlab-backup)**: Herramienta integrada en GitLab para crear respaldos completos de datos.
- **Bind mount**: Montaje directo de una ruta del host en el contenedor (menos recomendado que volumenes Docker).

## C

- **Container**: Instancia ejecutable de una imagen Docker, aislada del host.
- **Container Registry**: Registro de imagenes Docker integrado en GitLab.

## D

- **Docker**: Plataforma para crear, desplegar y ejecutar aplicaciones en contenedores.
- **Docker Compose**: Herramienta para definir y ejecutar aplicaciones multi-contenedor con un archivo YAML.

## G

- **Gitaly**: Servicio gRPC que ejecuta operaciones Git en GitLab.
- **gitlab.rb**: Archivo principal de configuracion de GitLab Omnibus.
- **gitlab-secrets.json**: Archivo de secretos que contiene claves de encriptacion (critico para restauracion).
- **gitlab-ctl**: Comando de administracion de GitLab Omnibus (status, restart, reconfigure, tail).

## H

- **HTTPS**: Protocolo HTTP con capa de seguridad TLS/SSL.

## L

- **Let's Encrypt**: Autoridad certificadora gratuita que emite certificados SSL/TLS automaticamente.

## O

- **Omnibus**: Metodo de empaquetado de GitLab que incluye todos los componentes necesarios.

## P

- **PostgreSQL**: Sistema de base de datos relacional usado por GitLab.
- **Puma**: Servidor de aplicaciones Ruby que maneja peticiones web en GitLab.

## R

- **Redis**: Almacen de datos en memoria usado para cache, colas y sesiones.
- **Root**: Usuario administrador por defecto en GitLab CE.

## S

- **Sidekiq**: Procesador de trabajos asincronos en GitLab.
- **SMTP**: Protocolo para envio de emails (notificaciones de GitLab).
- **SSL/TLS**: Protocolos de seguridad para comunicaciones en red.
- **Swap**: Espacio en disco usado como memoria virtual cuando la RAM se agota.

## V

- **Volumen Docker**: Mecanismo de persistencia de datos gestionado por Docker, independiente del ciclo de vida del contenedor.
