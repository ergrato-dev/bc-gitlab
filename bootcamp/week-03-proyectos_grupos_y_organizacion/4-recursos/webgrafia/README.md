# Webgrafia — Semana 03

- [GitLab Project Settings Documentation](https://docs.gitlab.com/ee/user/project/settings/)
  Documentacion oficial de todas las opciones de configuracion de proyectos: general, CI/CD, visibilidad, features.

- [GitLab Groups Documentation](https://docs.gitlab.com/ee/user/group/)
  Guia completa de grupos: creacion, subgrupos, transferencia, configuracion compartida.

- [GitLab Permissions and Roles](https://docs.gitlab.com/ee/user/permissions.html)
  Matriz completa de permisos para cada rol (Guest, Reporter, Developer, Maintainer, Owner) en todos los recursos.

- [Protected Branches](https://docs.gitlab.com/ee/user/project/protected_branches.html)
  Documentacion oficial de ramas protegidas: configuracion, wildcards, code owners, approval rules.

- [CODEOWNERS Syntax Reference](https://docs.gitlab.com/ee/user/project/codeowners/reference.html)
  Guia de sintaxis del archivo `.gitlab/CODEOWNERS` para asignar propietarios de codigo.

- [GitLab API — Projects](https://docs.gitlab.com/ee/api/projects.html)
  Endpoints REST para crear, leer, actualizar y eliminar proyectos via API.

- [GitLab API — Groups](https://docs.gitlab.com/ee/api/groups.html)
  Endpoints REST para gestion de grupos, subgrupos y membresias via API.

- [Conventional Commits Specification](https://www.conventionalcommits.org/en/v1.0.0/)
  Formato estandar para mensajes de commit (`feat:`, `fix:`, `docs:`, etc.) usado en el bootcamp.
