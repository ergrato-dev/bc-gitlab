# Glosario — Semana 04

## A

- **Assignee**: Persona asignada para resolver un issue o revisar un MR.
- **Approval**: Aprobacion formal de un MR por un reviewer autorizado.

## B

- **Board (Issue Board)**: Tablero Kanban que muestra issues organizados en columnas por label, milestone o assignee.

## C

- **Code Review**: Proceso de revision de codigo donde otros desarrolladores examinan los cambios antes del merge.
- **Confidential Issue**: Issue visible solo para miembros del proyecto con permisos Reporter o superiores.

## D

- **Draft MR (o WIP)**: Merge Request marcado como trabajo en progreso que no puede ser mergeado.
- **Diff**: Representacion visual de las diferencias entre dos versiones de un archivo.

## F

- **Fast-forward Merge**: Tipo de merge que simplemente mueve el puntero de la rama sin crear un commit adicional.

## G

- **GitLab Flow**: Flujo de trabajo que integra issues, branches, MRs y ramas de ambiente (staging, production).

## I

- **Issue**: Unidad de trabajo que rastrea tareas, bugs, features o mejoras en GitLab.
- **Issue Board**: Ver Board.

## L

- **Label**: Etiqueta categorizadora aplicable a issues y MRs (ej: ~bug, ~feature, ~frontend).

## M

- **Merge Request (MR)**: Solicitud para fusionar cambios de una rama origen a una rama destino.
- **Merge Commit**: Commit adicional creado al fusionar dos ramas con `--no-ff`.
- **Milestone**: Agrupacion temporal de issues y MRs con un objetivo comun (sprint, release, version).

## Q

- **Quick Action**: Comando inline (ej: `/assign`, `/label`, `/close`) que ejecuta acciones automaticamente en issues y MRs.

## R

- **Review Summary**: Resumen final del reviewer indicando si aprueba, solicita cambios o solo comenta.

## S

- **Squash and Merge**: Estrategia que combina todos los commits de una rama en un solo commit al mergear.
- **Suggested Change**: Propuesta de cambio de codigo que el autor puede aceptar con un click en el MR.

## T

- **Template**: Plantilla predefinida para estandarizar descripciones de issues (`issue_templates/`) o MRs (`merge_request_templates/`).

## W

- **Weight**: Estimacion de complejidad de un issue (escala 1-9).
- **WIP (Work In Progress)**: Ver Draft MR.
