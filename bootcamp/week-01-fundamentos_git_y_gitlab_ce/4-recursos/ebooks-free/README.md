## Pro Git (2ª Edición) — Scott Chacon y Ben Straub

**URL**: [git-scm.com/book/es/v2](https://git-scm.com/book/es/v2)
**Formatos**: Web (gratis), PDF, EPUB, Mobi
**Idioma**: Español (traducción completa) e Inglés

El libro oficial de Git. Escrito por uno de los creadores de GitHub y mantenido por la comunidad. Es la referencia definitiva para cualquier cosa relacionada con Git.

**Capítulos recomendados para esta semana**:

| Capítulo | Tema | Corresponde a |
|----------|------|---------------|
| 1 — Inicio | Qué es Git, instalación, configuración inicial | Lección 01 |
| 2 — Fundamentos de Git | Comandos básicos: init, clone, add, commit, log, diff | Lección 01 |
| 3 — Ramificaciones en Git | Branches, merge, rebase, flujos de trabajo | Lección 02 |
| 10 — Funcionamiento interno | Objetos Git, refs, packfiles (opcional, avanzado) | Curiosidad |

> 💡 Para esta semana enfócate en los capítulos 1, 2 y 3. Los capítulos 4-7 son muy relevantes para semanas posteriores.

## Git Internals — Scott Chacon

**URL**: [github.com/pluralsight/git-internals-pdf](https://github.com/pluralsight/git-internals-pdf/raw/master/drafts/peeled.pdf)
**Formato**: PDF (100 páginas)
**Idioma**: Inglés
**Nivel**: Avanzado / Curioso

Un PDF que explica cómo funciona Git internamente: el directorio `.git/`, los objetos (blobs, trees, commits, tags), las referencias (refs), el índice, los packfiles. No es necesario para el bootcamp, pero si eres de los que necesitan saber **por qué** funciona así, es fascinante.

## GitLab Handbook

**URL**: [handbook.gitlab.com](https://handbook.gitlab.com/)
**Formato**: Web (siempre actualizado)
**Idioma**: Inglés

La documentación interna de GitLab Inc. sobre cómo trabajan como empresa. Está completamente público. Incluye:
- Sus flujos de trabajo con Git
- Cómo hacen code review
- Sus convenciones de CI/CD
- Guías de liderazgo, comunicación y producto

**Para esta semana**: Sección [GitLab Workflow](https://handbook.gitlab.com/handbook/engineering/workflow/) — cómo el propio equipo de GitLab usa GitLab.

## The DevSecOps Playbook — GitLab

**URL**: [about.gitlab.com/resources/ebook-devsecops/](https://about.gitlab.com/resources/ebook-devsecops/)
**Formato**: PDF (requiere email para descarga)
**Idioma**: Inglés
**Nivel**: Intermedio

Ebook oficial de GitLab sobre DevSecOps: integrar seguridad dentro de los pipelines de CI/CD en lugar de hacerlo al final. Relevante para las semanas 5-10 del bootcamp.

## 🎯 Lectura Mínima para Aprobar la Semana

Si el tiempo es escaso, lee al menos:

1. **Pro Git Capítulo 1**: Secciones 1.1 y 1.3 (qué es Git y configuración básica)
2. **Pro Git Capítulo 2**: Secciones 2.1 a 2.5 (flujo básico: init, add, commit, push, pull)
3. **Pro Git Capítulo 3**: Sección 3.1 y 3.2 (qué es una rama y cómo hacer merge)

Tiempo estimado: ~90 minutos de lectura activa.
