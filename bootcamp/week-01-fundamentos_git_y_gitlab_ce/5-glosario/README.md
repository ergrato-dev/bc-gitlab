# Glosario — Semana 01

## A

- **Add**: Comando de Git que agrega archivos al staging area.

## B

- **Branch (rama)**: Linea independiente de desarrollo en Git.

## C

- **Clone**: Copia local de un repositorio remoto.
- **Commit**: Snapshot del estado del proyecto en un momento especifico.
- **Commit Message**: Mensaje descriptivo que acompana a un commit.

## D

- **DevOps**: Practica que une desarrollo (Dev) y operaciones (Ops).

## F

- **Fast-forward**: Tipo de merge donde Git simplemente mueve el puntero.

## G

- **Git**: Sistema de control de versiones distribuido.
- **GitLab CE**: GitLab Community Edition, version gratuita y open source.

## I

- **Issue**: Ticket para seguimiento de tareas, bugs o mejoras.

## M

- **Merge**: Fusion de una rama en otra.
- **Merge Request (MR)**: Solicitud de fusion de codigo en GitLab.

## P

- **Pipeline**: Conjunto de stages y jobs automatizados en CI/CD.
- **Pull**: Obtener cambios del repositorio remoto.
- **Push**: Enviar cambios locales al repositorio remoto.

## R

- **Rebase**: Operacion que reaplica commits sobre otra base.
- **Repository (repo)**: Almacen de archivos con seguimiento de cambios.
- **Remote**: Referencia a un repositorio remoto.

## S

- **SSH**: Protocolo seguro para autenticacion con Git.
- **Stage/Staging**: Area de preparacion antes del commit.
